# Weekly Knowledge Base Update

Review resolved nightly issues from the past week, evaluate triage accuracy,
and propose updates to the triage knowledge base via a PR.

**Budget: 40 tool calls max. Target: finish in under 30 tool calls. Batch aggressively.**

## Context Variables

Injected via the workflow:
- `REPO` — the repository (owner/name), e.g. `qcom-ai-hub/ai-hub-models-internal`
- `TETRACODE_REPO` — issue tracker, e.g. `qcom-ai-hub/tetracode`
- `LOOKBACK_DAYS` — how many days to look back (default: 7)

## Step 1: Collect Data (~6 tool calls)

Run these THREE calls in the SAME message (parallel):

**Call A** — nightly issues from the past week:
```
gh issue list --repo $TETRACODE_REPO \
  --search "[QAIHM Nightly] created:>=$(date -d '$LOOKBACK_DAYS days ago' +%Y-%m-%d)" \
  --label "ai-hub-models" \
  --state all --limit 20 \
  --json number,title,state,createdAt,closedAt,labels
```

**Call B** — merged PRs from the past week:
```
gh pr list --repo $REPO \
  --search "is:merged merged:>=$(date -d '$LOOKBACK_DAYS days ago' +%Y-%m-%d)" \
  --limit 50 \
  --json number,title,author,mergedAt,body,files
```

**Call C** — read all 4 KB files in one message:
```
cat .claude/triage/error-patterns.md .claude/triage/teams.md \
    .claude/triage/runtime-guide.md .claude/triage/examples.md
```

Then, for each nightly issue (max 2 calls — batch issue numbers):
```
gh api repos/$TETRACODE_REPO/issues/{num1}/comments --jq '[.[] | {author: .user.login, body: .body[0:2000]}]'
gh api repos/$TETRACODE_REPO/issues/{num2}/comments --jq '[.[] | {author: .user.login, body: .body[0:2000]}]'
```

From the comments, identify:
- **Agent comments** — contain "Breeze AI Nightly Analysis" or "Generated by Breeze AI"
- **Human follow-ups** — corrections, context, or "this was fixed by PR #X"

Also check each issue's **final labels and assignee** — these are the ground truth for
whether the agent's triage suggestion was correct (compare agent's suggested team vs actual label/assignee).

## Step 2: Evaluate + Discover (~6 tool calls)

Cross-reference issues with PRs. Match by:
- PR body references issue number
- PR title mentions the failing model name
- PR merged within 3 days of issue creation

For each nightly issue that had an agent triage comment, score:

| Score | Criteria |
|-------|----------|
| CORRECT | Suggested team AND root cause both right |
| PARTIAL | Team right but cause wrong, or cause right but team wrong |
| INCORRECT | Both wrong |
| MISSED | No agent analysis posted |

Build a summary table in your working memory (don't write it yet):
```
| Issue | Agent Team | Actual Fixer | Agent Cause | Actual Fix PR | Score |
```

Then identify KB updates needed:
- **New error patterns**: errors not matched by any existing pattern in `error-patterns.md`
- **Misrouted patterns**: existing rules that led to wrong team suggestion
- **New examples**: correct triage of non-obvious cases (add to `examples.md`)
- **Team changes**: fixers not listed in `teams.md` (add, never remove)

If no updates are needed (all triage correct, no new patterns), skip to Step 4 with
a "no updates" PR that just reports the accuracy score.

## Step 3: Apply Updates (~8 tool calls)

1. Create branch:
   ```
   git checkout -b auto/kb-weekly-update-$(date +%Y-%m-%d)
   ```

2. Edit the KB files using the Edit tool. Rules:
   - Only ADD or MODIFY — never delete without strong evidence
   - Keep edits minimal and targeted
   - Flag uncertain additions with "(low confidence — verify)"
   - For new error patterns: add to the appropriate confidence section
   - For new examples: append after the last numbered example
   - For misroute corrections: update the routing rule AND add an anti-pattern example

3. Commit + push in ONE sequence:
   ```
   git add .claude/triage/
   git commit -m "Weekly KB update: {N} new patterns, {M} corrections

   Triage accuracy: X/Y correct ({pct}%)
   {one-line per notable change}"
   git push -u origin auto/kb-weekly-update-$(date +%Y-%m-%d)
   ```

## Step 4: Create PR (~3 tool calls)

```
gh pr create --repo $REPO \
  --title "[Auto] Weekly triage KB update — $(date +%Y-%m-%d)" \
  --body-file /tmp/pr-body.md
```

Write `/tmp/pr-body.md` first with this format:
```markdown
## Weekly Triage Knowledge Base Update

**Period:** {start_date} to {end_date}
**Nightly issues reviewed:** {count}
**Triage accuracy:** {correct}/{total} ({pct}%)

### Accuracy Report
| Issue | Agent Team | Actual Fixer | Score |
|-------|-----------|-------------|-------|
| ... | ... | ... | ... |

### Changes Made
- {bulleted list of what changed and why, with links to issues/PRs}

### Misroutes Corrected
- {list, or "None this week"}

> Auto-generated by Breeze AI weekly KB update agent.
> Please review before merging.
```

## Rules

- **40 action limit** (each tool call = 1 action) — if approaching 35 actions, wrap up immediately (commit what you have)
- NEVER remove existing error patterns or team entries — only add or modify
- Always create a PR for human review
- Batch `gh` calls — never loop over issues one-by-one
- If no nightly failures this week → create a minimal PR noting "No failures to review"
- If tetracode access fails → fall back to reading commit comments on $REPO
- Keep PR description under 10,000 characters
- Include links to original issues and fixing PRs for verification
